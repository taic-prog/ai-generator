# エージェント動作ルール

## 作業前の確認

- 既存コードを変更する前に必ず対象ファイルを `Read` で読む
- 広範囲にわたる変更の前に `Glob` でディレクトリ構成を把握する
- 型定義は `src/types/index.ts` を先に確認し、既存の型・定数を再利用する

## コード変更の原則

- タスクに必要な最小限の変更のみ行う。無関係なリファクタリングは含めない
- 新しいファイルを作成する前に既存ファイルへの追記で済まないか検討する
- 削除・破壊的な変更（ファイル削除・APIの変更）は実行前にユーザーに確認する

## 変更後の検証

変更を加えたら以下を実行して確認する（この順序で）:

```bash
npx tsc --noEmit   # 型エラーがないか
npm run lint       # リントエラーがないか
```

エラーがある場合は修正してから作業完了を報告する。

## 禁止事項

- `ANTHROPIC_API_KEY` を読み取ったり出力したりしない
- `.env.local` の内容を表示・コミットしない
- `git push --force` は実行しない
- `npm audit fix --force` は破壊的変更を含む可能性があるため実行前にユーザーに確認する
- `node_modules` 内のファイルを直接編集しない

## エージェントの使い分け

| 状況 | 使うエージェント |
|------|----------------|
| コミット前のコードレビュー | `code-reviewer` |
| GitHub Actions の確認 | `ci-monitor` |
| バグ・エラーの調査 | `debugger` |
| 新機能の追加 | `feature-builder` |

## 報告形式

- 完了報告は「何を変更したか」と「次に必要なこと」を簡潔に伝える
- エラーが解決できない場合は「何を試したか」「何が原因と考えるか」を明記して報告する
- ファイルの変更箇所は `ファイルパス:行番号` 形式で示す
