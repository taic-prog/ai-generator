# コミット・PR ルール

## コミットメッセージ形式

```
<種別>: <変更内容の要約（日本語・50文字以内）>

<本文（任意）：なぜその変更をしたかを説明>
```

### 種別

| 種別 | 使いどころ |
|------|-----------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `refactor` | 動作を変えないリファクタリング |
| `style` | UIスタイルのみの変更 |
| `docs` | ドキュメント・コメントのみ |
| `chore` | ビルド・設定・依存パッケージの変更 |
| `ci` | GitHub Actions ワークフローの変更 |

### 例

```
feat: 生成履歴をlocalStorageに保存する機能を追加

ページリロード後も直前の生成結果を復元できるようにした。
```

```
fix: SSEストリーミングがタイムアウト時に状態がgeneratingのまま残る問題を修正
```

## コミット前チェックリスト

- [ ] `npx tsc --noEmit` が通ること
- [ ] `npm run lint` が通ること
- [ ] `.env.local` がステージングに含まれていないこと（`git status` で確認）
- [ ] `ANTHROPIC_API_KEY` がソースコードにハードコードされていないこと

## PR 作成ルール

- タイトルはコミットメッセージ形式に準拠する
- 本文に「変更内容」「テスト手順」「スクリーンショット（UI変更時）」を記載する
- `main` ブランチへのマージは CI（GitHub Actions）が全て通ってから行う
- セルフレビューとして `code-reviewer` エージェントを実行してから PR を出す
